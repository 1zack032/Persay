Security

How we keep your messages and data secure.

End-to-End Encryption

All messages on Menza are protected with AES-256 end-to-end encryption. This means only you and your intended recipients can read your messages‚Äînot us, not hackers, not anyone else.

üîê How It Works

When you send a message, it's encrypted on your device before being sent. Only the recipient's device has the key to decrypt it. Even if someone intercepts the message in transit, they can't read it.

Encryption Standards

  • AES-256: Industry-standard symmetric encryption for messages
  • PBKDF2: Secure key derivation with salt
  • TLS 1.3: Encrypted transport layer for all connections
  • HTTPS: All web traffic is encrypted

What We Cannot Access

Due to end-to-end encryption, we cannot read or access:

  • ‚ùå Message content
  • ‚ùå Shared files and media
  • ‚ùå Shared notes content
  • ‚ùå Your secret phrases

Account Security

Password Protection

  • Passwords are hashed using secure algorithms
  • We never store plain-text passwords
  • Failed login attempts are rate-limited

Session Security

  • Secure session tokens
  • Automatic session expiration
  • Ability to sign out all devices

Infrastructure Security

  • Secure servers: Hosted in SOC 2 compliant data centers
  • DDoS protection: Protection against distributed attacks
  • Regular updates: Systems are patched and updated regularly
  • Access control: Strict employee access policies

Security Best Practices

To maximize your security on Menza:

  1. Use a strong, unique password
  2. Don't share your login credentials
  3. Keep your device and browser updated
  4. Use auto-delete for sensitive messages
  5. Sign out when using shared devices

Reporting Security Issues

Found a security vulnerability? Please report it responsibly to security@menza.app. We take all reports seriously and will respond promptly.
